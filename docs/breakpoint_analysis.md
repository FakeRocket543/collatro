# Collatro 斷點分析：學生中斷點 × 爛 LLM 卡點

## 流程總覽

```
[1] ingest (NER) ─┐ 並行
[2] decompose (LLM) ┘
        ↓
[3] fuse (LLM agent)
        ↓
[4] enrich (Wikipedia/Wikidata API)
        ↓
[5] retrieve (搜尋引擎)
        ↓
[6] diff (LLM 比對)
        ↓
[7] render (Playwright 截圖)
```

---

## 🧑‍🎓 學生中斷點

### 步驟 1 — NER 斷詞
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| ckip-transformers 沒裝好 | 高 | `pip install` 失敗（PyTorch 版本衝突） |
| MLX 環境變數沒設 | 中 | 靜默 fallback 到 jieba，學生不知道 |
| 輸入太短（一句話） | 低 | 關鍵詞為空，後續全部空轉 |

**學生心理**：「裝不好就算了」→ 直接放棄第一步

### 步驟 2 — LLM decompose
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| 沒有 LLM server 在跑 | 極高 | `ConnectionError: 無法啟動 LLM` |
| mlx-lm 第一次下載模型 3GB+ | 高 | 卡在下載 10 分鐘，學生以為當機 |
| 學校網路擋 HuggingFace | 中 | 下載失敗 |

**學生心理**：「老師這個跑不動」→ 全班卡住等老師處理

### 步驟 3 — fuse
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| 步驟 2 已經失敗 | 高 | claims 為空，fuse 沒東西可合 |
| LLM 回傳非 JSON | 中 | fallback 機械合併（品質差但不會 crash） |

### 步驟 4 — enrich (Wikipedia)
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| 學校防火牆擋 Wikipedia API | 中 | timeout，每個實體等 10 秒 |
| 簡體實體名查不到中文維基 | 高 | `found: False`，Wikidata 也查不到 |
| 實體太泛（如「革命」「主義」） | 高 | 查到不相關的頁面 |

**學生心理**：「為什麼查不到普渡制药？」→ 因為中文維基用「普渡製藥」（繁體）

### 步驟 5 — retrieve (搜尋)
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| DuckDuckGo 被學校網路擋 | 中 | 搜尋結果為空 |
| 關鍵詞太爛（虛詞） | 高 | 搜到不相關結果 |
| DDG HTML 結構改版 | 低 | regex 抓不到結果，靜默回空 |

**學生心理**：「搜到的東西跟文章無關啊」

### 步驟 6 — diff
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| 證據為空（步驟 5 失敗） | 高 | 全部 `insufficient` |
| LLM 回傳格式錯 | 中 | verdict 顯示 `?` |

### 步驟 7 — render
| 中斷原因 | 機率 | 症狀 |
|----------|------|------|
| Playwright 沒裝 / chromium 沒下載 | 高 | `playwright install` 要另外跑 |
| 學校電腦沒權限裝 chromium | 中 | 權限錯誤 |

**學生心理**：「圖片出不來，這作業怎麼交？」

## 🤖 爛 LLM 卡點（3B 小模型 / 量化模型）

### decompose — 拆解主張
| 失敗模式 | 機率 | 後果 |
|----------|------|------|
| 回傳不是 JSON，夾雜解釋文字 | 極高 | `json.loads` 失敗，fallback 為整篇當一則主張 |
| JSON 格式對但欄位缺失 | 高 | `keywords` 為空 → 搜尋無效 |
| 把整篇文章複述一遍當「主張」 | 中 | 主張太長，搜尋關鍵詞無意義 |
| 只抓到 1-2 則主張（遺漏大量事實） | 高 | 查核覆蓋率低 |
| 幻覺：捏造文章沒說的主張 | 低 | 查核結果 mismatch 但其實是 LLM 的錯 |

### fuse — 融合
| 失敗模式 | 機率 | 後果 |
|----------|------|------|
| 回傳非 JSON | 中 | fallback 機械合併（可接受） |
| 沒有補英文名 | 高 | 搜尋效果跟不 fuse 一樣 |
| 把 NER 噪音原封不動保留 | 中 | 「主義」「革命」還是會被拿去搜 |

### diff — 比對
| 失敗模式 | 機率 | 後果 |
|----------|------|------|
| 永遠回 `match`（不敢說不一致） | 高 | 失去查核意義 |
| 永遠回 `insufficient`（懶得判斷） | 高 | 學生看到全黃燈 |
| 幻覺比對：主張證據說了 X 但其實沒有 | 中 | 誤導學生 |
| verdict 和 ner/numbers 矛盾 | 中 | 說 match 但列了 mismatch 項目 |

## 🔥 最致命的組合（連鎖失敗）

```
學校網路慢 + 3B 模型
  → 步驟 2: decompose 回傳垃圾 JSON
  → 步驟 3: fuse fallback，關鍵詞還是爛的
  → 步驟 5: 搜尋結果不相關
  → 步驟 6: diff 全部 insufficient
  → 步驟 7: 生成一堆黃燈圖片
  → 學生結論：「這工具沒用」
```

## 💡 建議防禦措施（優先序）

| # | 措施 | 解決什麼 | 工作量 |
|---|------|----------|--------|
| 1 | **預載模式**：老師課前跑 `collatro --preflight` 檢查所有依賴 | 步驟 1/2/7 的安裝問題 | 小 |
| 2 | **離線 cache**：第一次跑完存結果，第二次秒出 | 課堂 demo 不卡 | 小 |
| 3 | **OpenCC 簡→繁**：搜尋前自動轉換實體名 | 步驟 4/5 簡體查不到 | 小 |
| 4 | **英文 fallback 查詢**：fuse 補的英文名優先拿去搜 | 步驟 5 搜尋品質 | 已做(fuse) |
| 5 | **LLM 輸出驗證器**：檢查 JSON schema，不合格重試一次 | 步驟 2/3/6 的格式問題 | 中 |
| 6 | **降級模式**：LLM 全掛時用純 NER + 規則比對跑完 | 無 LLM 環境 | 中 |
| 7 | **Web UI**：老師機跑 server，學生用瀏覽器操作 | 所有安裝問題 | 大 |
